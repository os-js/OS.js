Configuring and building

You can find all the configuration files in src/conf directory.

The files are loaded in alphabetical order (that is why they are prefixed with a number) and merged togheter into one final map.

It is highly encouraged that you write your own files instead of modifiying the provided ones (because they may be affected by updates).

I provide a example in the repository.

If you make any changes, issue this command to rebuild configuration files (not needed if you run "grunt watch"): 

grunt config

Setting\Getting configuration via Grunt

You can also set (and get) configration via Grunt. 

grunt config:get:client.Connection.RootURI

grunt config:set:client.Connection.RootRUI:/custom/path

grunt config # rebuild config files

Building OS.js

OS.js has two build directories.

dist

This is the production build of OS.js. All files have been compiled into single files (and can be compressed) to increase loading speed. This will however make it harder to debug.

To build simply do: 

grunt

Or alternatively: 

grunt dist

dist-dev

This is what you should use when developing or testing changes. It will load all files directly from source (with exception of config and theme files).

This command is a shortcut to perform all tasks that dist-dev requires: 

grunt dist-dev
